ChatView Demo MFE

Project Overview

ChatView Demo is a Microfrontend (MFE) that provides a chat interface similar to Microsoft Teams. It serves as a demonstration of chat functionality that can be integrated into larger applications.

Purpose

  • Displays conversations in a familiar, modern chat layout
  • Supports both individual and group conversations
  • Provides a responsive, intuitive user experience
  • Can be integrated into other applications as a microfrontend

Core Functionality

Chat List (Left Panel)

  • Display all available conversations
  • Show conversation name, last message preview, and timestamp
  • Indicate unread message count with badges
  • Search/filter conversations

Chat Content (Right Panel)

  • Display messages for the selected conversation
  • Distinguish between own messages and messages from others
  • Show sender information and timestamps
  • Real-time typing indicator

CLAUDE.md ‚Äî vesser-chatview-demo

Project Overview

React/TypeScript microfrontend (MFE) providing a Teams-like chat interface. Uses Module Federation via @module-federation/vite and integrates with FusionKit for auth/logging.

Commands

npm start              # Dev server on port 4200
npm run build          # TypeScript compile + Vite build
npm run preview        # Preview production build
npx tsc --noEmit       # Type check without emitting
npx eslint src/        # Lint source files

Project Structure

src/
  Bootstrap.tsx        # MFE entry point (Module Federation)
  components/          # Reusable UI components
  views/               # Page-level components
  interfaces/          # TypeScript type definitions
  services/            # MockChatService, BotServiceRegistry
  hooks/               # useChatService
  assets/css/          # globals.css ‚Äî all CSS variables here
docs/                  # PROJECT_PLAN.md, IMPLEMENTATION_PLAN.md, plans/

Key Conventions

  • CSS Modules for component styling (*.module.css)
  • All colors MUST be defined in src/assets/css/globals.css
  • Views are page-level, components are reusable UI elements
  • MFE exposes ./bootstrap via remoteEntry.js
Known Issues & Solutions

Page Flickering on Chat Selection

Root Cause: selectedChatId in useEffect dependency array caused effect to re-run on every chat selection. Solution: Remove selectedChatId from dependencies.

Avatar "?" Placeholder Flash

Root Cause: React render batching caused messages from Chat A to render with Chat B's participant data. Solution: Add message.chatId === chat.id filter to message rendering.

Architecture Decisions

Bots as First-Class Users

Bots have isBot: true in the User interface ‚Äî not a separate Chat property. Any chat can have zero, one, or multiple bots as participants. Consistent with industry standards (Slack, Discord, Teams).
