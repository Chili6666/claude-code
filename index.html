02Phasen-Driven Development

Start · Implement · Verify · Stop

Every phase follows the same pattern. Claude starts with context, implements within the defined scope, verifies autonomously — and stops. A phase never runs into the next.

① Start

Load context

Read the phase file, reference CLAUDE.md, define a clear goal. What is being implemented? Which files are touched?

② Implement

Stay in scope

Claude works strictly within the defined scope. Deviations are documented, not silently implemented. No scope creep.

③ Verify

Automatic verification

Claude runs build, linter, and tests autonomously. If something fails: analyze error, fix root cause, re-run.

Build ausführen Linter ausführen Tests ausführen
⚠ Stopp

Explicitly pause

Claude reports: phase complete. The next phase only starts after confirmation by the developer.

④ Commit

Review & Commit

Developer reviews manually, runs /commit for structured commit message. Save phase summary under docs/plans/.

Manuelle Kontrolle Screenshot /commit ausführen
⑤ Reset

Clear context

Run /clear before the next phase. Fresh start, full focus, no old overhead in the context window.

/clear ausführen Kontext ist jetzt leer

★Kernvorteil

Der docs/-Ordner ist nicht nur Dokumentation — er ist das Gedächtnis des Projekts. Claude liest ihn, statt die Codebasis zu re-analysieren.

Best Practice Principle · Claude Code Workflow
01

Kein Re-Analyse-Overhead bei neuen Features

Claude findet alle relevanten Informationen strukturiert im docs/-Ordner. Neues Feature? Kurzer Blick in CLAUDE.md, IMPLEMENTATION_PLAN und die aktuelle Phase-Summary — fertig.

schnellere Iteration weniger Token-Verbrauch konsistentere Outputs
02

Erweiterungen folgen einem klaren Schema

Neue Entwickler und zukünftige Claude-Sessions verstehen das Projekt sofort — Conventions, bekannte Probleme und Architekturentscheidungen sind explizit dokumentiert.

Onboarding in Minuten Architektur bleibt konsistent
03

Phasen-Stops ermöglichen echte Kontrolle

Claude stoppt am Ende jeder Phase — du überprüfst, steuerst nach und gibst bewusst das Go für den nächsten Schritt. Kein unkontrollierter Vorlauf in eine falsche Richtung.

volle Kontrolle frühzeitiges Feedback weniger Rework

04 Slash Commands

Available Commands

Install any command once via npx — it is then available as a slash command in every project.

Command Category Description
/commit Git Guided commit workflow with prefix selection and optional push.
/organize-imports Code quality Sort & group imports in .ts/.tsx files with section comments.
/check-memoization React Detect unnecessary useMemo, useCallback, and React.memo usage.
/start-interviewing Planning Interview in plan mode and generate a structured plan document (project or feature).
/plan-phases Planning Generate a structured implementation plan with phases from an existing project or feature plan document.

05 Rules

Available Rules

Rules are applied automatically to matching files — no manual invocation needed. Install once via npx.

Rule Applies to Description
security *.ts · *.tsx · *.js · *.jsx Prevents hardcoded secrets, insecure dependencies, and OWASP vulnerabilities.
typescript *.ts · *.tsx Enforces import ordering, type safety, and consistent code quality in TypeScript projects.
npx github:Chili6666/claude-code rules add security
npx github:Chili6666/claude-code rules add typescript
